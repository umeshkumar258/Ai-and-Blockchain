# Contributing

Thanks for contributing to this project.

## Development Workflow

1. Fork the repository or create a feature branch.
2. Install dependencies with `npm run install-all`.
3. Run the backend with `npm run dev:backend`.
4. Run the frontend with `npm run dev:frontend`.
5. Test your changes locally before opening a pull request.

## Pull Request Guidelines

- Keep changes focused and easy to review.
- Update documentation when behavior changes.
- Avoid committing secrets such as `.env` files or private keys.
- Use clear commit messages that describe the change.

## Code Style

- Keep components and API handlers readable and small.
- Prefer descriptive variable names.
- Add comments only where they help explain non-obvious logic.
